A guide to creating an isolated Python environment for this project

Why Use a Virtual Environment?

A virtual environment is an isolated space for your Python project. It allows you to install specific versions of packages for one project without affecting other projects on your system. This is essential for avoiding dependency conflicts, like the ones encountered during this project's setup.

Step 1: Install the Correct Python Version

This project has dependencies that require a specific version of Python to function correctly. You must have Python 3.10 installed.

The fastest way to install it on Windows is using the winget command-line tool. Open PowerShell or Command Prompt and run:

winget install Python.Python.3.10

Step 2: Create the Virtual Environment

Once Python 3.10 is installed, navigate to your project's root directory in your terminal. Then, run the following command to create a new virtual environment named .venv using Python 3.10.

py -3.10 -m venv .venv

Note: This command explicitly tells the Python launcher (py) to use version 3.10 to create the environment. This is the most critical step to ensure compatibility.

Step 3: Activate the Virtual Environment

Before you can use the environment, you must "activate" it in your terminal session. This tells your terminal to use the Python interpreter and packages inside the .venv folder.

.venv\Scripts\activate

You will know it's active because your command prompt will change to have a (.venv) prefix.

Step 4: Install Project Dependencies

With the virtual environment active, you can now install all the project's required packages using the requirements.txt file. This will install them into the isolated environment, not your global Python installation.

pip install -r requirements.txt

Step 5: Deactivating the Environment

When you are finished working on the project, you can leave the virtual environment by simply running the deactivate command.

deactivate

The (.venv) prefix will disappear from your terminal prompt, and you will return to your system's default Python environment.
